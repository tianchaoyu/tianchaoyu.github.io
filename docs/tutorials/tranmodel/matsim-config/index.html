    想要Matsim正常运行，至少需要一个基础网络，一个出行需求，在这个基础上，再加上一个配置文件。除此以外，可以添加其他附加和可选输入数据。

    1、Matsim数据配置基础

    MATSim 使用多个文件来存储用于模拟的不同类型的数据。典型输入数据集：

    • config.xml：Matsim的配置选项
    • network.xml：道路网络描述
    • population.xml：出行需求，agent对象及日出行计划
    • facilities.xml：活动点相关位置信息
    • transitSchedule.xml：公交站点位置及公交服务信息
    • transitVehicles.xml：公共交通服务的车辆的描述

    • counts.xml：来自真实世界计数站的每小时数量以进行比较。

    • 上述文件概述了在使用MATSim时通常会遇到的文件，但并非总是需要所有文件。非常简单的模拟只能使用配置文件和网络描述以及包含代理计划的人口来运行。对于附加功能，例如对于公共交通的模拟，可能需要额外的文件。

    注意：一些文件，尤其是人口和网络文件（population.xml，network.xml或facility.xml），可能会变得非常大。为了节省空间，MATSim 支持以压缩格式读取和写入数据。 MATSim 为此使用 GZIP 压缩。因此，在许多情况下，文件名具有附加后缀tt .gz，如 population.xml.gz。 MATSim 会根据文件名自动检测文件是否已压缩或应写入压缩文件。

    2、配置文件示例

    2.1、配置选项文件：Configuration：config.xml

    配置文件，通常简称为config.xml，在用户和 MATSim 之间建立连接。它包含影响模拟行为方式的设置列表。所有配置参数都是简单的参数名称和参数值对。参数被分组到逻辑组中。例如，有一个组具有与控制器相关的设置，例如迭代次数，或者有另一个组具有与模拟相关的设置，例如模拟的结束时间。 可用参数列表和有效参数值可能因版本而异。尽管我们试图保持稳定，但由于软件的变化，尤其是新功能的变化，设置可能会发生变化。要获取当前可用的所有可用设置的列表，请运行以下命令：

    java -cp matsim.jar org.matsim.run.CreateFullConfig fullConfig.xml

    此命令将创建一个新的配置文件fullConfig.xml，其中包含可用参数的完整列表及其默认值。这样可以轻松查看可用的设置。要使用和修改某些设置，可以将具有相应参数的行复制到特定于要模拟的场景的配置文件中，并在该文件中修改参数值. 示例如下：

    <?xml version="1.0" ?>
<!DOCTYPE config SYSTEM "http://www.matsim.org/files/dtd/config_v1.dtd">
<config>

    <module name="network">
        <param name="inputNetworkFile" value="example/network.xml" />
    </module>

    <module name="plans">
        <param name="inputPlansFile" value="example/population.xml.gz" />
    </module>

    <module name="controler">
        <param name="outputDirectory" value="./output/" />
        <param name="firstIteration" value="0" />
        <param name="lastIteration" value="10" />
    </module>

    
</config>

    2.2、网络设置文件：network.Xml

    网络文件描述了agent（或车辆）可以在其上移动的基础设施。网络由nodes和links组成（在图论中，这些通常称为vertices 和 edges。每个元素都有一个标识符id。节点由 X 和 Y 坐标值描述。link有更多属性：from和 to属性引用节点并描述网络的几何形状。其他属性描述了网络与流量相关的方面.

    • length:link的长度，单位通常为米
    • flow capacity：link上每小时能通过的标准车的数量
    • freespeed:自由流速度，车辆通过link的最大速度，单位是m/s;
    • lanes:link上的车道数

    • modes:link上允许的速度，ex:modes=”car,bike,taxi”请注意，所有链接都是单向的。如果一条道路可以双向行驶，则必须使用交替的to和from属性定义两个链接.因此，网络可以看作是一个有向图。

#### 2.3、Matsim的出行需求设置：population.xml
MATSim 的旅行需求描述了智能体的一天出行计划。所有智能体通常是population，因此文件名为population.xml。在 MATSim 中也常用plans.xml， 因为人口文件本质上包含一个日计划列表。人口出行需求主要内容包括：

- The population contains a list of persons.
- Each person contains a list of plans.
- Each plan contains a list of Activities and Legs.

每人只有一个计划被标记为 selected。每个agent的选定计划是由移动模拟Mobsim执行的计划。在重新计划阶段，可能会将不同的计划标记为被选中。计划可以包含分数作为属性。在移动模拟执行计划后，在评分阶段计算分数并将其存储在计划中。每个计划中的活动和分支列表描述了每个agent的计划行动。活动具有分配的类型，并且具有 - 除了一天计划中的最后一个活动 - 定义的结束时间（有一些例外情况，活动具有持续时间而不是结束时间。此类活动通常由路由自动生成算法，因此不在本指南中描述）。为了描述活动发生的位置，通过给定 x 和 y 属性值来为活动分配坐标，或者分配一个link，描述从哪个链接可以到达活动。由于模拟需要链接属性，因此在缺少属性并且仅给出 x 和 y 坐标值或任何活动的情况下，控制器会为给定坐标计算最近的链接。
分支leg描述了agent如何计划从一个位置到下一个位置。每条支线leg都有一個可选属性，travel_time，他描述了该段出行的预期出行时间。对于模拟的分支leg，它必须包含一条路线。路线的格式取决于出行的交通方式。对于小汽车出行，路线列出了agent必须按给定顺序行驶的link，而对于公交出行，必须给定公交站点的位置以及预期交通服务的信息。
agent在前一个活动（或leg）结束后直接开始一个分支。根据mode的不同，移动模拟可能会以不同的方式处理agent。默认情况下，移动仿真很好地支持汽车和公交线路。如果 mobsim 遇到它不知道的模式，它默认为转移（teleportation），在这种情况下，agent从模拟现实中移除，并且在leg的预估出行时间过去后，重新插入agent的目标地点。

- 出行需求示例：

```html
<?xml version="1.0" encoding="utf-8"?>
<!DOCTYPE population SYSTEM "http://www.matsim.org/files/dtd/population_v5.dtd">
<population>
    <person id="1">
        <plan selected="yes" score="93.2987721">
            <act type="home" link="1" end_time="07:16:23" />
            <leg mode="car">
                <route type="links">1 2 3</route>

            </leg>

            <act type="work" link="3" end_time="17:38:34" />
            <leg mode="car">
                <route type="links">3 1</route>

            </leg>

            <act type="home" link="1" />
        </plan>

    </person>

    <person id="2">
        <plan selected="yes" score="144.39002">
            ...
        </plan>

    </person>

</population>

    人口数据格式是 MATSim 中最核心的数据结构之一，一开始可能有点让人不知所措。幸运的是，刚开始，只需知道它的一小部分。根据需要可以做下列简化：

    • Each person needs exactly one plan.
    • The plan does not need to be selected or have a score.
    • Activities can be located just by their coordinates.
    • Activities should have a somewhat meaningful end-time.
    • Legs only need a mode, but no routes.

    当开始模拟时，MATSim 的控制器将加载这样的文件，然后自动将最近的链接分配给每个活动，并为每条腿计算合适的路线。这使得快速入门变得容易。

    2.4、公交仿真模拟

    为了在 MATSim 中模拟公共交通，需要两个额外的输入文件：一个描述时间表，即线路、路线、停靠站点和出发时间。另一个描述了为线路服务的车辆：它们是大公共汽车、小型公共汽车、火车还是轻轨车辆，并描述了每辆车可以运送多少乘客。 公共交通车辆本身的描述可以分为两部分：第一部分，必须描述车辆类型，说明这种车辆可以运送多少乘客（注意，“车辆”一词实际上可以指多辆车辆，例如应将具有多节车厢的火车指定为具有大量座位的长车）。在第二部分中，必须列出实际车辆。每辆车都有一个标识符，并且是先前指定的车辆类型。 (示例文件:ptVehicles.xml)显示了这样一个文件的示例，描述了一种车辆类型和该类型的两种车辆。

    公共交通时间表，通常仅称为transitSchedule.xml，是一个相当复杂的文件。它包含有关停靠设施（这些可以是公共汽车站、火车站或其他停靠点）和交通服务的信息.在第一部分中，需要定义停靠设施，为每个停靠设施提供坐标、标识符和对网络中链接的引用。该站点只能由在该指定路段上行驶的车辆提供服务。或者，可以指定停靠点的名称，以及当运输车辆在停靠点等待时是否有其他车辆被阻塞。最后一个属性对建模很有用，例如公共汽车站的区别，一个公共汽车站有一个海湾，而在另一个车站，公共汽车必须停在实际道路上。
在停靠设施之后，描述了公交线路、它们的路线和时刻表。这是一个分层数据结构：每一行可以有一个或多个 routes，每条线路都有一个route profile、一个 network route和一个departures列表。示例文件：schedule.xml有一个简约但完整的运输时间表的例子。
每条公交线路都必须有一个唯一的 ID。每条公交路线都有一个 ID，该 ID 在该线路中必须是唯一的，因此相同的路线 ID 可以用于不同的线路。 transportMode描述了线路在网络中的哪些link上运行（实际上，目前尚未强制执行。在模拟中可以让公共汽车在火车链路上运行。将来可能会强制执行）.
routeProfile描述了这条路线服务的站点，而route本身描述了运输车辆的司机必须沿着的网络中的一系列链接（因此通常称为network route)。注意完整的路线，即车辆行驶的所有路段，必须在路线中列出，而不仅仅是停靠点所在的路段。所有指定的停靠点都应该按照指定的顺序沿着这条路线发生。每个给定的时间偏移量routeProfile中的 stop 描述了与实际出发时间的相对时间偏移。如果一辆公共汽车要在早上 7 点出发，并且停靠站 2 的离开偏移为00:03:00，则必须是读到公共汽车预计在 07:03 在特定站点出发。路线配置文件中的所有站点都必须定义一个出发偏移量，除了最后一个。所有站点，除了第一个，可以选择定义一个到达偏移量. 这对于停站几分钟的大型火车最有用在车站帮助路由算法找到正确时间的连接服务，即火车的预期到达时间。
作为公交路线描述的最后一部分，应给出出发清单。每个出发点都有一个 id，它在路线内必须是唯一的，并给出了指定路线配置文件的第一站的出发时间。此外，出发指定应使用哪辆车运行服务。该车辆必须在上述公交车辆列表中定义。

    2.6、单位和使用约定

    2.6.1、单位，Units

    MATSim 试图对实际单位做出尽可能少的假设，但在某些地方，没有任何假设是无法做到的。一般来说，MATSim 期望相似的值（例如所有距离）在任何使用它们的地方都使用相同的单位。下面简要概述了最重要的（预期的）单元。 -距离 距离单位最常用于link的长度。它们应该以与坐标系使用的相同单位来指定。这允许 MATSim 使用简单的三角测量，例如使用节点的坐标来计算直线距离。由于大多数通常使用的投影坐标系（参见 CoordinateSystems}）使用米作为距离单位，这是 MATSim 中最常用的距离单位。

    • 时间虽然 MATSim 在多个地方支持hour、minute、second表示，但在内部它使用seconds作为默认时间单位。这意味着例如link的速度必须以每秒距离指定，通常为m/s。 需要注意的是评分参数，MATSim期望的是每小时的值。这是因为大多数行为参数（如时间值）通常是每分钟或每小时估计的，并且秒的相应值非常小，因此容易配置错误。

    • 标识符MATSim 大量使用标识符，短 Id。此 Id 可以是任意字符串，但以下情况除外： Id 不应包含任何空格（包括制表符、换行符等）或逗号，因为这些字符通常用于在 Id 列表中将不同的 Id 相互分隔。

    • 坐标系统：Coordinate Systems在正确的坐标系中准备数据， 在多个输入文件中，您需要指定坐标，例如对于网络的节点。强烈建议不要使用 WGS84 坐标（即 GPS 坐标，或任何其他类型的球面坐标；西-东方向的坐标范围为 -180 到 +180，南-90 到 +90 -北方向）。 MATSim 需要在代码的几个地方计算两点之间的距离。球坐标之间距离的计算非常复杂并且可能很慢。相反，MATSim 使用简单的毕达哥拉斯定理，但这要求坐标位于笛卡尔坐标系中。因此，强烈建议使用笛卡尔坐标系，最好是距离单位对应于一米的坐标系。 在某些地方，MATSim 需要知道您的数据所在的坐标系。您有多种方法来指定您使用的坐标系。最简单的一种是使用所谓的“EPSG 代码”。大多数常用的坐标系都得到了标准化和编号。EPSG 代码唯一标识一个坐标系，可以直接被 MATSim 使用。作为替代方案，MATSim 还可以解析所谓 WKT 格式的坐标系描述。由于 WKT 格式更容易出错，因此建议尽可能使用 EPSG 代码。 要为您的坐标系找到正确的 EPSG 代码（例如，对于 UTM 区域之一），网站 http:http://www.spatialreference.org 非常有用。在此网站上搜索您的坐标系，)例如对于“WGS84 UTM Zone 8N”（对于北半球 UTM Zone 8），可以找到匹配坐标系及其 EPSG 代码的列表。 对于某些操作，MATSim 必须知道您的数据所在的坐标系。例如，某些分析可能会创建要在 Google Earth 中可视化的输出，其中坐标需要转换回 WGS84。您的数据使用的坐标系可以在配置文件中指定：

    <module name="global">
  <param name="coordinateSystem" value="EPSG:32608" />
</module>

    2.7、数据复用

    来自 PTV Visum 或 EMME 等其他建模工具的数据可以部分重复使用。通常只需很少的工作就可以转换网络拓扑。对于 Visum，http://matsim.org/javadoc/org/matsim/visum/package-summary.html部分代码可以将网络转换为 MATSim 的格式。但请注意，这只会转换网络拓扑（节点的位置，以及具有有限属性集的链接）。由于其他建模工具的工作方式不同（例如，使用速度-流量曲线来描述链接，BPR 函数是众所周知的函数之一），MATSim 可能有不同的解释（例如，link的容量是 MATSim 中的硬限制，与其他工具中的部分功能相比），导致在初始转换后需要手动编辑。 复用需求数据通常更复杂。如果您有基于agent和基于活动的需求生成过程的输出，您可以将数据agent逐个转换为 MATSim 格式。如果只有起点-目的地矩阵或吸引力矩阵形式的聚合数据可用，则无法简单转换为复杂的agent计划，并且必须使用其他数据源重新生成需求。
