    MATSim 没有任何易于使用的图形界面。相反，它使用文本文件进行配置，并使用命令行界面开始模拟。本章向您展示如何在许多不同的计算环境中启动 MATSim 模拟。

    1、Matsim基础环境

    1.1、软件基础：

    • MATSim 在任何安装了 Java(java Standard Edition) (SE) 环境的机器上运行。
    • java环境下载：https://www.oracle.com/java/technologies/downloads/> 注意下载低版本java,过高版本容易出bug,推荐java11;

    1.2、硬件环境

    • 较小的场景（例如教程中包含的示例），大型场景的（ 5%-10%示例）可以在普通电脑上运行，为了模拟大型场景（数十万个agent、具有数万个link和node的网络），可能需要具有大量内存 (RAM) 的高端计算机来将agent的数据保存在内存中。agent计划的描述和模拟输出可能会占用数GB的硬盘空间。为了存储多个场景的数据模拟运行的输出，可能需要大量的磁盘空间。MATSim 可以读取和写入压缩文件减少所需的磁盘空间量，但这方面仍然不应该被低估. MATSim 可以在重新规划阶段使用共享公共内存（“共享内存机器”）的多个 CPU 或 CPU 内核。为大量迭代运行大型场景可能需要几个小时，最多几天。因此，如果您计划模拟许多不同的场景，建议使用专用机器运行 MATSim。
    • 建议：
    • 要试用 MATSim：任何具有 1GB RAM 和 500MB 可用磁盘空间的现代笔记本电脑或台式电脑都应该是合适的。
    • 运行大型场景（100000+ 个代理，具有 50000+ 个链接的网络）： 一台至少有 4GB RAM 和 200GB 可用磁盘空间的高端台式计算机。
    • 运行许多大型场景，以便相互比较： \多台高端台式计算机或服务器，至少 4GB RAM，共享一个公共存储磁盘（至少 1TB）。

    • 可用磁盘空间的高数字是由于模拟在运行期间将大量数据写入磁盘的事实。对于分析，通常只需要最后一个版本的数据，并且可以删除早期迭代的数据，从而再次释放空间。### 1.3、matsim的并行化计算 运行 MATSim 仿真时，可以利用未使用的内核来加快仿真速度。现在有三种使用并行化来提高性能的方法，每种方法都可以单独打开或关闭。

    • 事件处理:ParallelEventsHandling，默认使用的事件处理程序与模拟运行在同一线程中。 因此，您可以通过将 parallelEventHandling模块添加到 MATSim \texttt{Config} XML 文件来打开并行事件处理，如下所示： 
        <module name="parallelEventHandling" >
      <param name="numberOfThreads" value="1" />
  </module>

      此模块中唯一需要的参数是numberOfThreads。此参数的值指定应分配多少线程（核心）来处理事件。

    • 一个额外的可选参数:estimatedNumberOfEvents， 可以针对模拟进行优化，从而加快运行速度。但是它的使用需要估计一次迭代中将发生的事件数量。 
        <param name="estimatedNumberOfEvents" value="5000000" >
    • 以下是一些需要考虑的提示和要避免的陷阱,获得的实际加速取决于许多因素：
    • 不要对执行处理程序的顺序做出任何假设（因为它们是并行执行的）。
    • 不要在一个处理程序中写入数据并在不同的处理程序中读取该数据,尽管这是可能的，但需要特别小心，因为为此需要线程之间的同步。此外，这可能导致速度下降。 \item 如果您的机器上只有一个内核可用，那么使用并行事件处理毫无意义。
    • 始终确保单 CPU 模拟需要一个线程。这意味着，如果您有 4 个内核，则可以将参数“numberOfThreads”设置为最大 3。 因为它不会使模拟更快（但在某些情况下可能会减慢速度）。
    • 处理程序的数量可以大于{numberOfThreads}。 在这种情况下，如果可能，自动为每个线程分配相同数量的处理程序。
    • 如果模拟非常慢（与事件处理相比），您将无法充分利用并行事件处理。## 2、Matsim的环境搭建

      2.1、java环境搭建

    • jdk软件下载：https://www.oracle.com/java/technologies/downloads/
    • maven插件下载：https://maven.apache.org/download.cgi
    • IntelliJ IDEA（或eclipse）编译器下载：https://www.jetbrains.com/idea/download/#section=windows> 注意 jdk和maven版本的适配性，否则容易出错，注意java的环境变量配置。推荐java11和maven-3.3.9

    2.2、matsim软件与安装

    • matsim下载链接：https://www.matsim.org/

    • 3种不同的使用方法：
    • 在编译器IDE下查看和使用源码
    • 单机版，使用GUI交互界面
    • 将matsim作为maven插件使用
    • github源码下载：https://github.com/matsim-org/matsim-libs
    • github样例测试：https://github.com/matsim-org/matsim-example-project

    2.3、matsim运行

    • mvnw.cmd clean package
    • java -jar matsim-example-project-0.0.1-SNAPSHOT.jar

    • 选择配置文件：example->scenarios->equil->config.xml
    • 点击start MATSIM
    • 查看结果文件源代码函数入口：matsim-libs/matsim/src/main/java/org/matsim/core/controler/Controler.java、
